# RGB 马赛克资产游戏设计文档

## 概述

这是一款以 Steam 为长期目标的像素资产游戏。玩家通过放置玩法挖掘真实 RGB 颜色，把这些颜色作为可消耗颜料填入像素画布，完成作品后进行鉴定。鉴定成功的马赛克作品会成为平台内唯一数字资产，并可以在类 Steam 市场中交易。

MVP 阶段先使用平台内资产库和平台内交易市场。资产模型从第一天开始预留 Steam Inventory 和 Steam Community Market 的映射能力，便于后续接入 Steam 库存与市场。

## 产品方向

已确认的产品路线是：**先做平台内资产和平台内交易，同时让资产模型兼容未来 Steam Inventory / Steam Market**。

这样可以避免第一版被 Steamworks、库存服务和市场审核流程卡住，同时不牺牲长期 Steam 化的方向。

## 核心循环

```text
放置挖矿产出 RGB 颜色和少量平台货币
-> 玩家把 RGB 颜色存入库存
-> 玩家手动把拥有的 RGB 颜色填到像素画布上
-> 玩家支付鉴定费并消耗作品使用的颜色
-> 系统判断该像素矩阵是否为平台内唯一
-> 鉴定成功后，作品成为可交易的平台资产
-> 资产进入排行榜和交易市场
-> 后续版本可映射为 Steam Inventory 物品
```

## MVP 范围

MVP 应聚焦于跑通一个完整的资产闭环。

| 模块 | MVP 设计 |
| --- | --- |
| 挖矿 | 放置产出真实 RGB 颜色和少量平台货币 |
| 颜色库存 | 相同 RGB 颜色按数量叠加 |
| 画布 | 第一版只开放 `16x16` |
| 创作 | 玩家手动使用库存 RGB 颜色填色 |
| 鉴定 | 消耗作品使用的颜色和平台货币鉴定费 |
| 唯一性 | `画布尺寸 + 完整 RGB 像素矩阵` 必须全平台唯一 |
| 资产 | 鉴定成功后生成唯一平台内资产 |
| 交易 | 鉴定后的资产可以在平台内市场挂售和购买 |
| 榜单 | 评分榜和价值榜 |
| Steam 预留 | 资产字段预留 Steam Inventory 映射数据 |

`32x32` 和 `64x64` 作为后续成长解锁，不进入第一版核心实现范围。

## RGB 颜色挖矿

RGB 颜色是真实资源，不是抽象道具。每次挖到的颜色表示为：

```text
RGB(R, G, B)
R/G/B 范围：0-255
```

相同颜色会在库存中叠加：

```text
RGB(255,0,0) x2
RGB(12,98,201) x7
RGB(0,0,0) x1
```

挖矿采用放置产出。玩家不需要持续点击，系统会随时间产出颜色。挖矿也会产出少量平台货币，用于降低新手参与鉴定和市场交易的门槛。

## 颜色稀有度

颜色稀有度由 RGB 属性和全平台供应数据计算，不手动硬设“普通 / 史诗 / 传说”等道具等级。

稀有度来源包括：

| 来源 | 示例 |
| --- | --- |
| 极端通道值 | `RGB(255,0,0)`、`RGB(0,0,0)`、`RGB(255,255,255)` |
| 高饱和度 | 霓虹色、强对比色 |
| 平台供应量低 | 全平台出现次数较少的颜色 |
| 首次发现 | 第一个挖到某个 RGB 的玩家可以获得发现者标记 |
| 特殊色域 | 纯色、灰阶、金属感颜色区间 |

稀有度系统应输出一个标准化的颜色稀有度分数，用于资产评分和未来 UI 筛选。

## 画布与创作

MVP 画布尺寸为 `16x16`，共 `256` 个像素。

创作规则：

1. 玩家只能使用自己拥有的 RGB 颜色。
2. 填充 1 个像素会占用 1 个对应 RGB 颜色。
3. 作品鉴定前可以撤销编辑，被占用的颜色会返还。
4. 作品鉴定成功后，使用的颜色会被永久消耗。
5. 鉴定成功的作品不可再编辑。
6. 后续加入更大画布时，不同尺寸应使用独立榜单。

## 鉴定与唯一性

鉴定会把完成的画布转化为平台内唯一资产。

鉴定成功必须同时满足：

1. 画布尺寸受系统支持。
2. 所有必要像素都有合法 RGB 值。
3. 玩家拥有足够的 RGB 颜色数量。
4. 玩家可以支付鉴定费。
5. 平台内不存在相同 `画布尺寸 + 完整有序 RGB 像素矩阵` 的已鉴定作品。

唯一性指纹：

```text
pixel_hash = hash(画布尺寸 + 有序 RGB 像素矩阵)
```

全平台唯一规则是严格规则：如果两张作品尺寸相同，并且每一个像素位置的 RGB 值都完全一致，那么只有第一张成功鉴定的作品可以成为资产。

## 资产模型

鉴定成功的作品是唯一核心可交易资产。

RGB 颜色只是创作材料，MVP 和 Steam 市场规划中都不开放交易。

核心资产字段：

```text
asset_id
creator_id
owner_id
canvas_size
pixel_matrix
pixel_hash
metadata_hash
created_at
certified_at
rarity_score
system_score
trade_score
final_score
last_sale_price
total_volume
trade_status
```

Steam 预留字段：

```text
steam_item_id
steam_marketable
steam_tradable
steam_inventory_status
```

未来链上预留字段：

```text
chain_id
contract_address
token_id
mint_tx_hash
```

内部 `asset_id` 始终是主记录。Steam Item ID 或链上 Token ID 只是外部映射，不替代平台内部资产记录。

## 交易市场

MVP 交易市场是使用平台货币的类 Steam 内部市场。

只有鉴定成功的唯一马赛克资产可以交易。

MVP 不交易：

1. RGB 颜色库存。
2. 未鉴定草稿。
3. 平台货币本身。

交易规则：

1. 资产持有人可以设置固定价格并上架作品。
2. 买家可以按挂售价直接购买。
3. 成交后资产所有权立即转移。
4. 每笔成交记录资产 ID、卖家、买家、价格、手续费和时间。
5. 平台收取交易手续费，初始设计可按 `5%`。
6. 成交数据进入价值榜，并参与最终评分中的交易分。

拍卖、报价、求购单不进入 MVP 范围。

## 货币经济

MVP 使用混合货币来源：

1. 放置挖矿产出少量基础货币。
2. 出售鉴定作品是主要财富来源。
3. 鉴定费和市场手续费作为货币回收机制。
4. MVP 不开放 RGB 颜色交易。

这样可以保持经济系统易理解，同时支持作品资产交易。

## 评分与榜单

最终评分由系统评分和交易表现组成：

```text
final_score = system_score * 0.9 + trade_score * 0.1
```

系统评分占主导，避免评分榜被交易刷榜完全控制。

建议系统评分组成：

| 项目 | 建议权重 |
| --- | ---: |
| 颜色稀有度 | 35% |
| 消耗颜色价值 | 20% |
| 色彩复杂度与多样性 | 15% |
| 画布完成度 | 10% |
| 首次鉴定或早期发现奖励 | 10% |
| 作者信誉 | 10% |

交易分只计算真实成交。建议输入项包括：

1. 最近成交价。
2. 历史成交总额。
3. 有效成交次数。
4. 持有人变化。

榜单：

| 榜单 | 排名依据 |
| --- | --- |
| 评分榜 | `final_score` |
| 价值榜 | 成交价、历史成交额和市场活跃度 |

后续加入更大画布时，应按尺寸拆分榜单，例如 `16x16`、`32x32`、`64x64`。

## 反刷规则

交易数据容易被操控，因此 MVP 必须有基础防刷机制。

初始反刷规则：

1. 挂售价不计入成交表现。
2. 只有真实完成的成交影响交易分。
3. 同一批账号之间短期重复交易会降权。
4. 异常高价成交可进入观察状态，审核前不计入评分。
5. 短时间频繁倒手会降低交易分贡献。
6. 交易分最多只占最终评分的 `10%`。

价值榜可以更直接反映市场热度，但评分榜必须主要由系统评分决定。

## Steam 集成路线

Steam 是长期目标，不作为 MVP 的前置依赖。

| 阶段 | 目标 |
| --- | --- |
| MVP | 平台内账号、资产库、库存、市场、评分 |
| Steam 准备 | 接入 Steam 登录和 Steamworks 基础配置 |
| Steam Inventory | 把已鉴定平台资产映射为 Steam Inventory 物品 |
| Steam Market | 在符合资格和审核要求后，让部分已鉴定作品具备市场交易能力 |

Steam 设计原则：

1. 只有鉴定成功的唯一作品才可能进入 Steam Inventory 或 Steam Market。
2. RGB 颜色继续作为平台内消耗资源。
3. 内部 `asset_id` 是权威记录。
4. Steam Item ID 是内部资产的外部映射。
5. 游戏不应承诺投资收益或现实世界利润。
6. Steam 市场资格、可交易性和可上架状态取决于 Steamworks 规则和审核。

## MVP 不包含内容

以下内容明确推迟：

1. Steam Community Market 真实接入。
2. NFT 铸造或链上转移。
3. RGB 颜色交易。
4. 拍卖、报价或求购系统。
5. 大于 `16x16` 的画布。
6. AI 相似度查重。
7. 真实货币支付。
8. 宣传资产具有投资属性。

## 实施阶段待定问题

以下问题进入实施计划阶段再确定：

1. 放置挖矿间隔和颜色掉落分布。
2. 颜色稀有度公式和评分归一化方式。
3. 鉴定费金额和成长曲线。
4. 市场手续费比例和货币回收平衡。
5. 草稿是否必须填满整个画布才能鉴定。
6. 作者信誉如何计算，且不鼓励刷信誉。
7. 如何展示像素预览和元数据，以兼容未来 Steam 库存展示。
