# RGB 马赛克资产游戏产品文档

## 1. 产品定位

RGB 马赛克资产游戏是一个以 RGB 色块为基础资源、以像素画创作为核心操作、以作品鉴定为资产化节点的网页游戏原型。

当前阶段目标不是上线完整经济系统，而是验证最小玩法闭环：玩家能获得颜色、消耗颜色创作、管理草稿、鉴定完整作品，并看到作品被锁定为唯一资产。

## 2. 当前阶段目标

当前阶段只做网页端前端 MVP。

核心目标：

1. 验证挖矿、库存、创作、鉴定的基础循环是否成立。
2. 验证画作生命周期是否清晰。
3. 验证色块消耗和返还规则是否直观。
4. 验证唯一资产概念是否能被前端表达清楚。
5. 为后续后端、市场和桌面客户端设计留下清晰边界。

## 3. 目标用户

第一阶段目标用户是内部验证者和早期玩法测试者。

他们需要回答的问题：

1. 挖矿获得颜色是否有持续反馈。
2. 色块库存是否容易理解。
3. 用库存颜色创作像素画是否顺手。
4. 作品完成后鉴定为资产是否有明确仪式感。
5. 草稿、已鉴定作品和内置作品的状态是否容易区分。

## 4. 核心玩法循环

```text
自动挖矿
-> 获得 RGB 色块
-> 色块进入库存
-> 新建或选择未鉴定作品
-> 消耗色块填充像素
-> 完成 16x16 画作
-> 发起像素扫描鉴定
-> 生成唯一资产记录
-> 已鉴定作品锁定
```

## 5. 当前功能范围

### 5.1 挖矿

挖矿是自动进行的放置行为。

规则：

1. 默认进入挖矿页。
2. 系统按固定间隔产出 RGB 色块。
3. 每次产出一个代表色。
4. 最近产出记录只展示 10 条。
5. 产出的颜色自动进入色块库存。

### 5.2 色块库存

色块库存用于展示玩家持有的 RGB 颜色。

规则：

1. 相同 RGB 颜色按数量叠放。
2. 未拥有代表色仍在图鉴中显示，数量为 0。
3. 色块具有稀有度。
4. 玩家可选择有库存的色块用于填色。
5. 库存数量为 0 的色块不能用于填色。

### 5.3 画作库存

画作库存用于管理所有作品。

规则：

1. 画作卡片只展示作品信息和鉴定状态。
2. 点击卡片只切换右侧作品详情。
3. 继续创作、鉴定、重命名、删除入口集中在右侧详情。
4. 已鉴定作品不显示继续创作入口。
5. 已鉴定作品不显示重命名和删除入口。

### 5.4 新建画作

新建画作用于创建玩家草稿。

规则：

1. 新建画作必须输入名称。
2. 空名称不能创建。
3. 创建成功后进入画布编辑。
4. 未鉴定作品最多 5 个。
5. 达到上限后不能继续新建。

### 5.5 画布创作

画布是固定 `16x16` 的像素创作区域。

规则：

1. 顶部点击画布页不会自动创建作品。
2. 必须新建作品或选择未鉴定作品后才能编辑。
3. 选择库存色块后点击像素填色。
4. 填色消耗 1 个当前颜色。
5. 覆盖已有像素时，消耗新颜色并返还旧颜色。
6. 已填相同颜色的像素不能重复消耗。
7. 每次填色自动保存。
8. 没有撤回和手动保存按钮。

### 5.6 作品重命名

重命名只对未鉴定作品开放。

规则：

1. 重命名入口位于作品详情右上角 `...` 菜单。
2. 双击标题不能重命名。
3. 已鉴定作品名称锁定。
4. 重命名同步库存卡片、详情和画布下拉列表。

### 5.7 删除草稿

删除只对玩家新建的未鉴定草稿开放。

规则：

1. 删除入口位于作品详情右上角 `...` 菜单。
2. 内置哥布林和皮卡丘不能删除。
3. 已鉴定作品不能删除。
4. 删除玩家草稿会返还已填色块。
5. 删除当前正在编辑的草稿后，画布回到未选择状态。

### 5.8 鉴定与资产

鉴定是把完整作品转为平台内资产的动作。

规则：

1. 未完成作品不能鉴定。
2. 完整作品可以发起鉴定。
3. 鉴定前先计算像素矩阵哈希。
4. 如果哈希已存在，不进入扫描并提示重复。
5. 如果哈希唯一，播放像素扫描动画。
6. 鉴定成功后生成资产记录。
7. 鉴定成功后作品状态变为已鉴定。
8. 已鉴定作品不可继续编辑、不可重命名、不可删除。

## 6. 当前数据模型

### 6.1 色块

字段：

1. RGB 值。
2. 数量。
3. 稀有度。

### 6.2 画作

字段：

1. 作品 ID。
2. 作品名称。
3. 状态：草稿、已鉴定、归档。
4. 宽度和高度。
5. 像素矩阵。
6. 更新时间。
7. 鉴定后的像素哈希。

### 6.3 资产

字段：

1. 资产 ID。
2. 关联作品 ID。
3. 作品名称。
4. 像素哈希。
5. 创建者。
6. 拥有者。
7. 鉴定时间。

## 7. 本地保存

当前使用 `localStorage` 保存前端状态。

保存内容：

1. 色块库存。
2. 画作列表。
3. 资产列表。
4. 挖矿总数。
5. 当前选中的作品。
6. 当前画布编辑作品。

不保存内容：

1. 当前页面。
2. 鉴定扫描弹窗状态。
3. 最近挖矿记录。

## 8. 成功标准

当前阶段成功标准：

1. 用户能在 5 分钟内理解挖矿、库存、画布、鉴定的关系。
2. 用户能独立新建画作并完成填色。
3. 用户能理解库存色块是可消耗资源。
4. 用户能理解鉴定后作品被锁定。
5. 用户能理解重复像素矩阵不能生成第二个资产。
6. 核心规则都有自动化测试覆盖。

## 9. 后续路线

建议路线：

1. 继续验证前端玩法闭环。
2. 抽离前端领域逻辑和存储接口。
3. 设计 Go 后端 MVP。
4. 把挖矿、防作弊、资产唯一性迁移到后端。
5. 增加资产详情页和资产库。
6. 增加平台内市场原型。
7. 重新评估桌面客户端方案。
8. 再评估 Steam 相关接入。

## 10. 风险和边界

当前版本不是线上经济系统。

风险边界：

1. 前端本地唯一性校验不能防作弊。
2. `localStorage` 不能作为正式存档方案。
3. 没有账号系统，不能区分真实用户。
4. 没有后端，不能保证全平台资产唯一。
5. 没有市场和支付系统，不涉及真实交易。
6. Steam 相关内容只是长期预留方向。
